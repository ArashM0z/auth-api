Layer 1 · per IP

Coarse flood control

Every request first passes an INCR against the caller's IP. A single address can't fire an unlimited storm at the API regardless of which account it targets. Health probes are exempt so a busy load balancer can't lock itself out.

Layer 2 · per username

Targeted lockout (this demo)

Login failures are counted per account. The slot is consumed before Argon2id runs — a concurrent burst of guesses is serialized by Redis INCR, so at most the cap can ever reach the hash. A correct password clears the counter.
